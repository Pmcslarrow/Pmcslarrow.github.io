  2. (0.5 points) Explain the difference between L1 and L2 regularization and say what advantage(s) each one confers. Put another way: explain when you might use one over the other.

  3. (1 point) Give some reasons why you think Dropout can mitigate the overfitting issue (useful readings: https://www.cs.toronto.edu/~hinton/absps/JMLRdropout.pdf)?

ANSWER Q1

  1. L1 encoruages parameters to be sparse (0) by penalizing high values of the l1 norm of the model parameters. L2 encourages parameters to be small but never actually 0 by penalizing high values of the l2-norm. These both use a lambda value that dictates by how much you regularize the data. For example, with L2 regularization, with a higher value for lambda, it will most intensly shift values closer to 0. They are both used as regularization techniques to reduce test error at the expense of an increase of training error.

  2. The textbook notes that the "sparsity property induced by L1 regularization has been used...as a feature selection mechanism" (Chp 7. 232). This means that certain weights are set to zero, so certain features can safely be ignored because they are less important for the task. L2 regularization is used when you don't want to ignore the effects of specific weights, but create a more stable/generalizable model that still uses the weights, they just have less of an affect rather than none.

  3. The reason that Dropout can mitigate against overfitting issues, is that it trains an "ensemble" of subnetworks. It essentially thins networks and then averages these predictions into a single generalization that helps reduce overfitting. If we think back to the lecture slides, we can get difference loss landscapes from different examples. By removing certain units from a network, we effectively adjust the loss landscape, in turn making our model more generalizable over the entire dataset, rather than overfitting on one landscape, but getting closer to the true landscape.

import torch
import numpy as np
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
from torch.utils.data import DataLoader, Dataset

class MultiplyDataset(torch.utils.data.Dataset):
    """MULTIPLY Dataset."""

    def __init__(self, num_examples, max_abs=1000):
        """create a dataset of the form x_1 * x_2 = y. The input x_1, x_2 is a
        pair of values drawn from the default range [-1000, 1000]. The output y is a scalar.

        PARAMETERS
        ----------
        num_examples    An integer determining how much data we'll generate
        max_abs         The largest absolute value a datapoint can have
        """
        self.length = num_examples

        # make a circular unit Gaussian and draw samples from it
        data = np.random.uniform(low=-1, high=1, size=(self.length, 2))

        data *= max_abs
        # figure out the label (i.e. the result of the multiplication)
        label = np.multiply(data.T[0],data.T[1])

        # turn it into a tensor
        self.data =  torch.tensor(data).to(dtype=torch.float32)
        self.label = torch.tensor(label).to(dtype=torch.float32)

    def __len__(self):
        return self.length

    def __getitem__(self, idx):
        return self.data[idx], self.label[idx]

class Net(nn.Module):
  def __init__(self):
    super().__init__()
    self.fc1 = nn.Linear(2, 1000)
    self.fc2 = nn.Linear(1000, 2048)
    self.fc3 = nn.Linear(2048, 1000)
    self.fc4 = nn.Linear(1000, 1)
    self.ac = nn.ReLU()

  def forward(self, x):
    x = self.ac(self.fc1(x))
    x = self.ac(self.fc2(x))
    x = self.ac(self.fc3(x))
    x = self.fc4(x)

testing_loss = torch.zeros((epochs, ))

for i in range(epochs):

  # Training loop
  epoch_loss = 0.0

  for x, y in train_dl:
    net.train()
    opt.zero_grad()

    output = net(x)
    loss = loss_func(output.squeeze(), y)
    epoch_loss += loss.item()

    loss.backward()
    opt.step()

  epoch_loss /= train_num_batches
  training_loss[i] = epoch_loss

  # Testing loop

  with torch.no_grad():
    for x, y in test_dl:
      pred = net(x)
      loss = loss_func(pred.squeeze(), y)
      epoch_loss += loss.item()
    epoch_loss /= test_num_batches
    testing_loss[i] = epoch_loss

best_training_loss = training_loss.min().item()

We're now going to think about how easy or hard it is to solve the general multiplication problem for real numbers: $y= x_1*x_2$, when $x_1,x_2 \in {\Bbb R}$ are not bounded to the limited range (-1000,1000).

Define a "simple" feed-forward neural network as one where each layer $l$ takes input from only the previous layer $l-1$. Let's assume our simple feed-forward network only uses "standard" nodes. We'll define a "standard" one as one that take a weighted sum $z=\mathbf{w}^T\mathbf{x}$ of inputs $\mathbf{x}$, given weights $\mathbf{w}$, and then applies a differentiable activation function $f()$ to $z$. Example "standard" nodes include ReLu, Leaky ReLU, Sigmoid, TanH, and the linear/identity function.

Define "correctly" performing multiplication as estimating $\hat{y}= x_1*x_2$ to 2 decimal places of precision (i.e. $ |\hat{y}-y| < 0.01)$. Here, $\hat{y}$ is the network's result and $y$ is the true answer.

Is it possible to make a "simple" feed-forward neural network that can correctly perform multiplication for any arbitrary pair $x_1,x_2 \in {\Bbb R}$? Support your answer. This doesn't have to be a mathematical proof, but you do have to clearly explain your reasoning.

Hint: Think about what a "standard" node calculates. Think about how you would implement multiplication using addition.

ANSWER Q4

Somewhat following what I argued in Q3, I would argue that it is NOT possible to make a "simple" feed-forward neural network that can correctly perform multiplication for an arbitrary pair in the reals. What I mean by this is that a standard linear transformation computes the weighted sum such that $z = w^Tx$ of inputs $x$ gives weights $w$. This comes out to be $w_1 * x_1 + w_2 + x_2 \dots w_n * x_n$. This makes it noticeable that it is only modifying the sum of weighted values and inputs, and in turn cannot learn x1 * x2 between inputs. Alone, we know that multiplication cannot be learned by a linear function because multiplication is a non-linear function. Therefore, we attempt to use "standard" nodes like "ReLU, Leaky ReLU, Sigmoid, and TanH". Even with these standard activation functions to add non-linearity to the model, I would still argue that with large values for x1 and x2, a large nonconvex space is created where our model parameters would have to memorize all of the possible combinations (basically) in order to accurately learn these. We also must notice that with larger numbers multiplied together, it could cause exploding or vanishing gradients within our network. It is very possible that such large values for the output, combined with a loss function such as MSE, would create an extremely large loss value that is largely skewing the gradients. As I read through chapter 8 of the textbook, it is clear that a large number of factors could be causing this error, it is hard to say which is the main cause, but it could largely be the weight initializations, large number of minima that have a "high cost in comparison to the global minimum" in the non-convex neural space we have created, or even our lack of using gradient clipping to mitigate against the vanishing and exploding gradient problem.

This is why when the decision space was between (-5 and 5), our model was able to achieve low scores for loss to solve the multiplication problem, but as we increase the number of our absolute value range, it would be increasingly impossible to model due to its large decision space, lack of complexity of the model, and infinite space of the real numbers.

import torch
import torch.nn as nn

class YouNet(nn.Module):
    def __init__(self,
                 kernel_1: tuple[int, int],
                 kernel_2: tuple[int, int],
                 stride_1: tuple[int, int] = (1, 1),
                 stride_2: tuple[int, int] = (1, 1)):

        mnist = self.conv1(x)
        imagenet = self.conv2(mnist)

class NoBatchModel(nn.Module):
  def __init__(self):
    super().__init__()
    self.fc1 = nn.Linear(28*28, HIDDEN)
    self.fc2 = nn.Linear(HIDDEN, HIDDEN)
    self.fc3 = nn.Linear(HIDDEN, HIDDEN)
    self.fc4 = nn.Linear(HIDDEN, 10)
    self.sig = nn.Sigmoid()

  def forward(self, x):
    batch_size, _, _, _ = x.size()
    x = x.view(batch_size, -1)

    x = self.sig(self.fc1(x))
    x = self.sig(self.fc2(x))
    x = self.sig(self.fc3(x))
    x = self.fc4(x)
    return x



class WithBatchModel(nn.Module):
  def __init__(self):
    super().__init__()
    self.fc1 = nn.Linear(28*28, HIDDEN)
    self.fc2 = nn.Linear(HIDDEN, HIDDEN)
    self.fc3 = nn.Linear(HIDDEN, HIDDEN)
    self.fc4 = nn.Linear(HIDDEN, 10)
    self.sig = nn.Sigmoid()
    self.b1 = nn.BatchNorm1d(HIDDEN)
    self.b2 = nn.BatchNorm1d(HIDDEN)
    self.b3 = nn.BatchNorm1d(HIDDEN)


  def forward(self, x):
    batch_size, _, _, _ = x.size()
    x = x.view(batch_size, -1)

    # Input --> Hidden layer 1
    x = self.fc1(x)
    x = self.b1(x)
    x = self.sig(x)

    # Hidden layer 2
    x = self.fc2(x)
    x = self.b2(x)
    x = self.sig(x)

    # Hidden layer 3
    x = self.fc3(x)
    x = self.b3(x)
    x = self.sig(x)

    # Hidden 3 --> Out
    x = self.fc4(x)
    return x

def train_model(model, device='cpu', lr=0.001):
  model.to(device)
  opt = optim.SGD(model.parameters(), lr=lr, momentum=0.9)
  step_count = 0

  # Initializing variables for loss and acc
  training_loss = torch.zeros((STEPS,))
  training_acc = torch.zeros((STEPS,))
  testing_loss = torch.zeros((STEPS // 20) + 1,)
  testing_acc = torch.zeros((STEPS // 20) + 1,)

  while step_count < STEPS:

    # TRAINING

    for idx, batch in enumerate(train_dataloader):
      model.train()
      if step_count >= STEPS:
        break

      opt.zero_grad()
      x, y = batch
      x = x.to(device)
      y = y.to(device)

      output = model(x)
      loss = LOSS_FUNC(output, y)

      preds = output.argmax(dim=1)
      acc = preds.eq(y).sum().item()/len(y)

      loss.backward()
      opt.step()

      training_acc[step_count] = acc
      training_loss[step_count] = loss.item()

      # VALIDATION EVERY 20 STEPS
      if (step_count % 20 == 0):
        model.eval()
        val_idx = step_count // 20
        with torch.no_grad():
            batch = next(iter(val_dataloader))
            x = x.to(device)
            y = y.to(device)

            output = model(x)
            loss = LOSS_FUNC(output, y)

            preds = output.argmax(dim=1)
            acc = preds.eq(y).sum().item()/len(y)

            testing_acc[val_idx] = acc
            testing_loss[val_idx] = loss.item()

      step_count += 1

      if (step_count % 5000 == 0):
        print("Step :", step_count)

  return training_loss, training_acc, testing_loss, testing_acc, model
