TeamsOfMen App


Summary

Our group focused on designing a serverless AWS architecture leveraging Lambda and API Gateway to create a clean, scalable, and modular design for the project. The primary goal is to develop a mobile application for Kip Ioane's TeamsOfMen healthy masculinity curriculum. Kip initially introduced this curriculum at Willamette University, where he taught players and staff about healthy masculinity and character development as tools for navigating everyday experiences, particularly in college. Over the past decade, Kip has expanded his vision beyond basketball, aiming to make this curriculum his full-time endeavor, helping other programs nationwide benefit from understanding and practicing healthy masculinity through daily self-reflection.


Note to Reader:

The backend portion of the application is primarily complete. The frontend is a work-in-progress, but the important details/infrastructure that the app is built around is complete, and described below in some detail.



AWS Lambda + API Gateway

We designed the backend using AWS Lambda in combination with API Gateway to achieve a cost-effective and scalable architecture. API Gateway handles the routing of requests, allowing Lambda functions to serve as our compute engine for more intensive and complex tasks. This modular setup minimizes costs while maximizing performance and scalability.


One of our key Lambda functions includes the capability to process and insert new coursework PDFs. We leverage the OpenAI API along with a strict JSON formatting scheme to extract and structure information from these PDFs efficiently. This data is then stored appropriately in our database or S3, based on retrieval needs.



Data Storage: RDS and S3

Our application utilizes Amazon RDS for structured data that requires quick retrieval and transactional consistency. RDS ensures reliable, scalable, and secure database operations.


For long-term storage and less frequently accessed files, we use Amazon S3. Within S3, we employ compression and decompression techniques to optimize storage space. This is particularly beneficial for large files, such as PDFs containing coursework information, to minimize overall storage costs.



Authentication

We have implemented a robust authentication system using JSON Web Tokens (JWT). JWT allows for secure and stateless communication by encrypting information within the token itself, reducing the need for repeated database queries. This design choice significantly minimizes cost and increases efficiency by limiting database interactions during authentication processes and API calls.



Complex Tasks and PDF Processing

To support complex backend tasks, such as inserting and managing new coursework PDFs, we use a combination of AWS services. Lambda functions are integrated with the OpenAI API to extract structured data from these PDFs using a JSON scheme. This ensures consistency and accuracy in how data is processed and stored across the system.



Summary of Key Features

  • AWS Lambda + API Gateway: Efficient request routing and scalable compute functions.
  • RDS: Quick retrieval of structured, transactional data.
  • S3: Long-term storage with compression to optimize space.
  • Authentication: JWT-based stateless security to minimize database trips.
  • PDF Processing: Automated extraction and storage using OpenAI API and structured JSON formatting.

This structured backend approach ensures seamless integration with the frontend as it is developed, providing a secure, scalable, and cost-effective solution.
